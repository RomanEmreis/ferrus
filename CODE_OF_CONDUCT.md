# Code of Conduct

## Our standard

This project aims to be a professional, respectful place for collaboration.

Participants are expected to:

- communicate clearly and respectfully;
- discuss technical disagreements in good faith;
- focus criticism on ideas, code, and behavior, not on people;
- avoid harassment, intimidation, discrimination, or personal attacks;
- respect the time and attention of maintainers and contributors.

## Unacceptable behavior

Examples of unacceptable behavior include:

- abusive or degrading language;
- harassment, threats, or sustained hostility;
- discriminatory remarks or exclusionary behavior;
- publishing private information about someone without permission;
- deliberately disruptive behavior in issues, pull requests, or discussions.

## Scope

This Code of Conduct applies in project spaces, including issues, pull requests, discussions,
and other communication channels used for project collaboration.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions
that violate this Code of Conduct. They may also temporarily or permanently restrict participation
when necessary to protect the project and its contributors.

## Reporting

If you experience or witness unacceptable behavior, report it privately to the project maintainers.
If the repository later adds a dedicated reporting channel, this document should be updated accordingly.

## Interpretation

Maintainers are expected to enforce this policy in good faith and with the goal of keeping
the project constructive, safe, and useful for collaboration.
